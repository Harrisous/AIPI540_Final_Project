# AIPI540_Final_Project
by Haochen Li

## Topic:
Stock market prediction using environment embedding and transformer.

## Themes:
Finance Technology

## Approaches:
- A naive approach (i.e. a mean model)
- A non deep learning approach using classical ML techniques
- A NN-based deep learning approach

## Deliverables:
1. working, interactive proof-of-concept application, which must be publicly accessible via internet for the week following submission. You may use any frameworks you wish to create your front-end, such as Flask, Streamlit, or Dash and you can deploy your model using any platform (ie GCP, Azure, AWS).

2. 10-minute maximum presentation (there is a hard stop!) submitted as a video link on Canvas (YouTube, WarpWire, Panopto). The presentation should include the following:
Problem you are addressing
Data source(s)
Review of relevant previous efforts and literature (if existing dataset)
Your model evaluation process & metric selection
Your modeling approach
Data processing pipeline
Models evaluated and model selected (must include at least 1 non deep learning model and 1 deep learning model - see details above)
Comparison to naive approach
Brief demo of your project
Results and conclusions
Ethics statement

3. Demo Day "Pitch" (maximum 3 minutes and 3 slides, there is a hard stop). The pitch should focus on a demonstration to a wider audience and should consist of:
Problem you are addressing/motivation (1 slide)
Brief overview of approach (1 slide)
Brief demo of your project
Results/conclusions/interesting findings (1 slide) Note: Since this is open to the public, we recommend not sharing anything that you don't want to be prior art. For the majority of projects, this should not be an issue. However, if you are planning to extend your idea in a startup let's chat about options.

4. Submitted link to a working, publicly accessible web or mobile interface of your application deployed on a cloud platform (ie GCP, AWS, Azure)
